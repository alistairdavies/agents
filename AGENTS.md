# Agent Instructions

 - Make use of project level agent instruction files and proactively keep them up to date when making changes.
 - Do not create commits on the users behalf.
 - Do not assume the users idea is the best approach. Suggest alternatives and challenge when best practices are not followed.
 - Use extended thinking liberally.


## Workflow

For non-trivial tasks, follow this pattern:

1. **Plan** - Understand the problem, explore relevant code, ask clarifying questions, outline approach
2. **Execute** - Implement in small, verifiable steps
3. **Verify** - Run tests, check types, validate the change works


## Skills
Make use of the skills available whenever they fit the context of the task.
